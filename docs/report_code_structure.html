Code Structure Report: Traffic Officers Productivity Project

Generated: March 16, 2026

Project Overview: This project studies overtime allocation among LADOT Traffic Officer IIs, estimating how informal trading networks and a seniority-based "wheel" system determine who works overtime shifts. It estimates a structural model of overtime participation, recovers officer valuations, and simulates counterfactual allocation mechanisms (auctions, random assignment). The codebase spans R and Stata across three main directories, plus a near-duplicate analysis_network/ directory.

3. Full Pipeline: Analysis (analysis/)

Stage 00: Data Preparation for Analysis

00_01_individual_productivity.R
Purpose: Recover individual officer safety productivity (traffic direction skill) using Bonhomme-style group decomposition. Officers assigned to OT shifts near special events; net out time/size FEs; solve linear system for individual contributions.
Inputs: mkdata/data/01_03_pre_network_90.csv, mkdata/data/00_01_collisions_with_mindist.rds
Outputs: analysis/data/00_01_individual_productivity.rds
↓
00_02_mk_estimation_sample.R
Purpose: Creates the central estimation sample. Reads 90-day network panel, drops post-injury recovery days, attaches productivity, holiday indicators, bereavement/FMLA flags from raw data. Constructs key analysis variables (opp_dist, suppliers_interacted, ot_rate).
Inputs: mkdata/data/01_06_panel_working.rds, analysis/data/00_01_individual_productivity.rds, raw anonymized data
Outputs: analysis/data/00_02_estimation_sample.rds — nearly every downstream file reads this
↓

4. The analysis_network/ Directory

This is a near-complete duplicate of analysis/ with 17 files that mirror files in analysis/. The key difference is that some files use a network-based definition of the estimation sample (using analysis_network/data/ instead of analysis/data/). It contains its own 00_01, 00_02, 02_00–02_06, 03_00–03_99 files. Additionally contains 00_03_event_studies.R and heatmap.R not found in main analysis.

7. Output Structure

All outputs go to analysis/out/:

  • analysis/out/tables/ — LaTeX tables (.tex): summary stats, estimation results, simulation comparisons, cartel impacts
  • analysis/out/figures/ — PNG figures: network plots, histograms, binscatters, equity-efficiency frontiers, event studies
  • mkdata/out/ — Intermediate diagnostics, CSVs, map figures
